# Repro-in-a-Box v2.0 🎁

**Find bugs. Freeze them. Ship them.**

Autonomous QA agent that finds bugs on your site, captures reproducible evidence, and ships them directly to your issue tracker.

## 🚀 Quick Start

```bash
# Install dependencies
npm install

# Build
npm run build

# Scan a website
repro scan https://your-site.com

# Or use npm scripts
npm run dev -- scan https://your-site.com
```

## 📦 What's Implemented (Week 1)

### ✅ Week 1: Complete Detection Suite

**Detector Framework**
- Base detector interface with lifecycle hooks (`setup`, `attach`, `scan`, `collect`, `cleanup`)
- Detector registry for managing multiple detectors
- Issue severity levels: `info`, `warning`, `error`, `critical`
- Issue categories: `javascript`, `network`, `assets`, `accessibility`, `performance`, `security`, `links`

**7 Working Detectors**

1. **JavaScript Errors** (`js-errors`)
   - Console errors (error, warn level)
   - Uncaught exceptions (thrown errors)
   - Unhandled promise rejections
   - Source URL and message extraction

2. **Network Errors** (`network-errors`)
   - Failed HTTP requests (4xx, 5xx)
   - Request timeouts
   - DNS failures
   - Connection refused errors

3. **Broken Assets** (`broken-assets`)
   - Broken images
   - Missing stylesheets
   - Failed scripts
   - Broken fonts
   - Any resource with HTTP ≥400

4. **Accessibility** (`accessibility`)
   - WCAG 2.1 Level A & AA violations via axe-core
   - Missing alt text on images
   - Insufficient color contrast
   - Missing form labels
   - Landmark/heading structure issues
   - Impact-based severity mapping (critical → critical, serious → error, moderate/minor → warning)

5. **Web Vitals** (`web-vitals`)
   - Core Web Vitals measurement
   - CLS (Cumulative Layout Shift): good <0.1, needs improvement 0.1-0.25, poor >0.25
   - INP (Interaction to Next Paint): good <200ms, needs improvement 200-500ms, poor >500ms
   - LCP (Largest Contentful Paint): good <2500ms, needs improvement 2500-4000ms, poor >4000ms
   - FCP (First Contentful Paint): good <1800ms, needs improvement 1800-3000ms, poor >3000ms
   - TTFB (Time to First Byte): good <800ms, needs improvement 800-1800ms, poor >1800ms
   - Only reports "needs improvement" or "poor" metrics

6. **Mixed Content** (`mixed-content`)
   - HTTP resources on HTTPS pages
   - Active mixed content (scripts, stylesheets) → CRITICAL
   - Passive mixed content (images, media) → WARNING
   - Security downgrade detection

7. **Broken Links** (`broken-links`)
   - Checks first 50 links on each page
   - HTTP 4xx/5xx detection
   - Network failures (timeouts, connection refused)
   - HEAD request optimization
   - Anchor text + target URL reporting

**CLI Framework**
- Commander-based CLI with `scan`, `validate`, `diff` commands
- Multi-page crawler with:
  - Configurable depth and page limits
  - Rate limiting (default 1000ms)
  - Same-domain filtering
  - URL pattern include/exclude
- Scanner orchestrator that runs all enabled detectors across pages
- JSON output for scan results
- Rich console output with color-coded severity

**Testing**
- Unit tests for detector registry (5/5 passing)
- Integration tests via CLI
- Comprehensive test page with 25+ issues
- Real-world validation (W3C demos, GitHub, example.com)
- Demo script showing all detectors in action

### 📋 Commands

#### `scan` - Detect issues on a site

```bash
repro scan <url> [options]

Options:
  -d, --max-depth <number>    Maximum crawl depth (default: 2)
  -p, --max-pages <number>    Maximum pages to scan (default: 10)
  -r, --rate-limit <ms>       Rate limit between requests (default: 1000)
  -o, --output <path>         Output file path (JSON)
  --no-headless               Run browser in visible mode
  --same-domain-only          Only crawl pages on the same domain (default: true)
```

Examples:
```bash
# Quick scan (single page)
repro scan https://example.com --max-depth 0

# Deep scan (multiple pages)
repro scan https://example.com --max-pages 50 --max-depth 3

# Save results
repro scan https://example.com --output results.json

# Watch the browser
repro scan https://example.com --no-headless
```

#### `validate` - Verify reproducibility (Coming in Week 3-4)

```bash
repro validate <scan-file>
```

#### `diff` - Compare scan results (Coming in Week 5)

```bash
repro diff <scan1> <scan2>
```
