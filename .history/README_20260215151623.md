# Repro-in-a-Box v2.0 🎁

**Find bugs. Freeze them. Ship them.**

Autonomous QA agent that finds bugs on your site, captures reproducible evidence, and ships them directly to your issue tracker.

## 🚀 Quick Start

```bash
# Install dependencies
npm install

# Build
npm run build

# Scan a website
repro scan https://your-site.com

# Or use npm scripts
npm run dev -- scan https://your-site.com
```

## 📦 What's Implemented (Week 1)

### ✅ Week 1, Days 1-3: Core Framework

**Detector Framework**
- Base detector interface with lifecycle hooks (`setup`, `attach`, `scan`, `collect`, `cleanup`)
- Detector registry for managing multiple detectors
- Three working detectors:
  - **JavaScript Errors**: Catches console errors, uncaught exceptions, unhandled promise rejections
  - **Network Errors**: Detects failed HTTP requests, timeouts, 4xx/5xx errors
  - **Broken Assets**: Finds broken images, stylesheets, scripts, and fonts

**CLI Framework**
- Commander-based CLI with `scan`, `validate`, `diff` commands
- Multi-page crawler with:
  - Configurable depth and page limits
  - Rate limiting
  - Same-domain filtering
  - URL pattern include/exclude
- Scanner orchestrator that runs detectors across pages
- JSON output for scan results

**Testing**
- Unit tests for detector registry (5/5 passing)
- Integration tests via CLI
- Demo script showing detectors in action

### 📋 Commands

#### `scan` - Detect issues on a site

```bash
repro scan <url> [options]

Options:
  -d, --max-depth <number>    Maximum crawl depth (default: 2)
  -p, --max-pages <number>    Maximum pages to scan (default: 10)
  -r, --rate-limit <ms>       Rate limit between requests (default: 1000)
  -o, --output <path>         Output file path (JSON)
  --no-headless               Run browser in visible mode
  --same-domain-only          Only crawl pages on the same domain (default: true)
```

Examples:
```bash
# Quick scan (single page)
repro scan https://example.com --max-depth 0

# Deep scan (multiple pages)
repro scan https://example.com --max-pages 50 --max-depth 3

# Save results
repro scan https://example.com --output results.json

# Watch the browser
repro scan https://example.com --no-headless
```

#### `validate` - Verify reproducibility (Coming in Week 3-4)

```bash
repro validate <scan-file>
```

#### `diff` - Compare scan results (Coming in Week 5)

```bash
repro diff <scan1> <scan2>
```
